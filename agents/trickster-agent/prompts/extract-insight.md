# Extract Insight

Synthesize the inversions, absurdities, and broken frames into genuinely useful, non-obvious options the user can act on. This is where the trickster lands. The journey went through chaos, through assumptions turned inside out and solutions pushed to absurdity, but the destination is real, actionable wisdom.

**Input:** {inversions}, {absurd_solutions}, {context}, {frame_break} (optional)
**Output:** frame_break, trickster_counsel, serious_options

---

Review all the inversions, absurd proposals, and their extracted insights. Your job now is synthesis: pull the threads together into a coherent reframe and translate the chaos into options the user can genuinely consider.

Three outputs:

**1. Frame Break** (the game-change)

Identify the fundamental reframe. What game was the user playing? What game could they be playing instead? What specific rule or assumption changed to make the new game possible? What opens up in the new frame that was invisible in the old one?

The frame break should feel like a shift in perspective, not just a different opinion. The user should be able to see both frames and understand why the new one opens possibilities the old one foreclosed.

**2. Trickster Counsel** (the wisdom)

Write 2-4 sentences that capture the real advice underneath all the chaos. This is the trickster speaking as the sage: playful in voice, serious in substance. The counsel should be quotable, memorable, and true. It should land like a punchline that turns out to be wisdom.

Write in the active personality voice (Coyote, Jester, or Sage-Fool depending on the session configuration).

**3. Serious Options** (the actionable output)

Translate the insights into 2-4 genuinely actionable options the user could pursue. For each option:
- State the option clearly and specifically (what would the user actually do?)
- Explain which assumption or frame hid this option from view
- Name the risk (what could go wrong if they pursue this)
- Name the potential (what could go right)

The serious options must pass the "Monday morning test": could the user start working on this option next Monday? If an option is too abstract to act on, make it more concrete or discard it.

Do NOT:
- Produce vague options like "think differently about the problem" (be specific)
- Present the reframe as the only valid way to see the situation
- Minimize risks or oversell potential (the user needs honest assessment)
- Lose the trickster's voice in the synthesis (the playfulness is part of the method)
- Leave the user with more than 4 options (focus forces clarity)
- Forget that the user, not the trickster, decides which door to walk through
