# Generate Absurd Solutions

Propose deliberately absurd solutions to the stuck situation. Each absurd proposal targets a specific invisible assumption, violating it as loudly as possible. The absurdity is not the destination. It is a controlled explosion designed to crack open the assumption and reveal the real insight hiding inside.

**Input:** {stuck_situation}, {invisible_assumptions}, {constraints}
**Output:** Array of absurd solutions, each with absurd_proposal, assumption_it_violates, why_its_absurd, real_insight_hidden_inside

---

For each invisible assumption, design one absurd solution that violates it dramatically. The absurdity should be memorable, specific, and targeted, not random.

Principles of productive absurdity:

1. **Targeted, not random**: Each absurd proposal must violate a specific assumption. "What if we hired a clown?" is random. "What if we made every meeting cost $100 per person per hour from each team's budget?" is targeted (it violates the assumption that meetings are free).
2. **Concrete, not vague**: "What if we did the opposite?" is too vague. "What if we gave customers the source code and let them fix the bugs themselves?" is concrete enough to analyze.
3. **Scalable insight**: The real insight should apply beyond the absurd framing. The absurd proposal is a thought experiment; the insight is a principle.
4. **Playful, not hostile**: The absurdity should make people laugh or at least raise an eyebrow. It should never mock the people involved in the situation.

For each absurd proposal:
- State the absurd solution clearly and concretely
- Name which invisible assumption it violates
- Explain briefly why it is absurd (what makes it impractical or impossible as stated)
- Extract the real insight hiding inside: what kernel of truth does this absurdity point toward? What would a practical version of this absurd idea look like?

The "real insight hidden inside" is the most important part. If you cannot find a real insight inside the absurdity, discard the proposal and generate a different one. Absurdity without insight is noise.

Generate 3-5 absurd proposals, each targeting a different invisible assumption. If fewer than 3 assumptions were surfaced, generate multiple absurd proposals per assumption, each revealing a different facet of insight.

Do NOT:
- Generate absurdity for entertainment without extractable insight
- Be cruel or mocking toward the people in the stuck situation
- Propose absurdities that involve harm, deception, or manipulation
- Present the absurd proposals as actual recommendations (they are vehicles for insight)
- Generate more than 5 proposals (each one must earn its place with a real insight)
