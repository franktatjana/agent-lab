# Reframe Problem

Challenge the current problem definition, test its assumptions, and generate genuinely different alternative frames. The most expensive organizational mistake is solving the wrong problem with excellence. This skill catches it before the investment.

## Prompts Used

1. **gather-signals** - Collect signals and context that inform the current problem definition
2. **challenge-frame** - Question the current problem definition, surface hidden assumptions, and generate genuinely different alternative problem definitions

## Workflow

[Gather Signals] -> organized signal inventory -> [Challenge Frame] -> assumption inventory with fragility assessment + alternative problem definitions with action implications

## When to Use

- The team has been working on a problem for months without progress, which may indicate the problem is defined wrong
- The obvious solution keeps failing, suggesting the problem definition misses the actual dynamic
- A powerful stakeholder defined the problem and nobody has questioned the frame since
- Different departments propose contradictory solutions, which often means they are solving different problems
- The team says "we know what the problem is" with too much confidence given the ambiguity of the situation

## What You Get

1. The current problem definition stated explicitly with its embedded assumptions surfaced
2. An assumption stress test ranking assumptions by fragility (most likely to be wrong first)
3. A frame diagnosis assessing whether this is the right problem, a partial problem, or possibly the wrong problem
4. 2-3 genuinely different reframes, each with its own signal support, action implications, and what it reveals that the current frame hides

## Example Input

"We have a retention problem. Our churn rate is 18% annually, up from 12% two years ago. The product team is building a loyalty program. Customer success is increasing touchpoints from quarterly to monthly check-ins. The CEO has framed this as 'we need to be stickier.' Meanwhile, our NPS has actually improved from 32 to 41 over the same period."

## Tips

- The NPS-churn divergence in the example above is a classic reframing signal. If customers like the product more but leave more, "retention problem" might be the wrong frame. "Pricing problem," "segment-fit problem," or "competitive positioning problem" could be the actual frame.
- Reframes are not word games. "Our retention problem" and "our customer satisfaction problem" are not different frames; they are different labels for the same frame. A genuine reframe changes which signals matter, which causes are relevant, and which solutions become visible.
- The hardest reframe to accept is the one that invalidates work already done. If the team has spent six months building a loyalty program and the reframe suggests the problem is pricing, the sunk cost of the loyalty program creates powerful resistance to the reframe. Name this dynamic explicitly.
