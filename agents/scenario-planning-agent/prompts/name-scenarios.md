# Name Scenarios

Give each scenario a memorable, evocative name and a distilled 2-3 sentence narrative. Names are the primary recall mechanism for scenario sets. A team that cannot remember the scenario names cannot use the scenarios in daily decision-making. The name should capture the essence of the world so vividly that hearing it immediately evokes the full narrative.

## When to Use

After the construct-worlds prompt has built four scenario narratives. This prompt takes the full narratives and distills each into a name and a capsule summary. The naming step is not cosmetic, it is the step that determines whether the scenarios live in organizational memory or die in a slide deck. Shell's most famous scenarios ("World of Internal Contradictions," "New Frontiers") persisted for decades because the names were evocative enough to trigger instant recognition.

## Prompt

You have four scenario worlds from a 2x2 matrix. Your job is to give each a name that captures its essence and a 2-3 sentence narrative that distills the full world into a memorable capsule.

**1. Naming Principles**
Each name must be:
- **Evocative**: triggers an image, feeling, or association. "Digital Archipelago" is evocative. "High Tech Low Regulation" is a label, not a name
- **Distinctive**: hearing any one name should not be confusable with the others. If two names could be swapped without confusion, they are not distinctive enough
- **Neutral**: the name should not telegraph whether the scenario is "good" or "bad." "Golden Age" and "Dark Winter" pre-judge the scenario and bias the team's analysis
- **Memorable**: one to three words. If the name needs explanation, it is too abstract. If it needs qualification, it is too vague

**2. Narrative Capsule**
For each named scenario, write 2-3 sentences that:
- Open with the most distinctive feature of this world
- Include the key tension or dynamic that defines life in this world
- Close with what this world means for actors in the strategic context

The capsule should be standalone: someone who reads only the name and capsule should understand what makes this world different from the other three.

**3. Name Set Coherence**
Review all four names together:
- Do they feel like they belong to the same exercise? They should share a register (all metaphorical, all descriptive, all narrative) rather than mixing styles
- Can you say all four names in one breath and have someone grasp the range of futures? "Walled Gardens, Open Frontier, Regulated Commons, Fragmented Bazaar" conveys the spread. "Good, Bad, Medium, Other" does not
- Would you use these names in a board presentation? If any name sounds silly, flippant, or confusing out of context, revise it

**4. Matrix Placement**
Present the four named scenarios on the 2x2 matrix with axis labels. The visual placement should make it immediately clear which combination of uncertainty endpoints each scenario represents.

Deliver: four names, four 2-3 sentence capsules, and the labeled 2x2 matrix. The output should be something the team can pin to a wall and reference for months.

## Tips

- Shell's scenario teams spend disproportionate time on naming because they know that scenarios without memorable names do not survive the meeting where they are presented. The name is the handle the organization grabs to use the scenario in real decisions.
- Avoid names that are too clever or too abstract. "Quantum Entanglement" might describe a world of interconnected systems, but if the CFO does not immediately grasp it, the name fails. Test names against the "can I explain this to someone in 10 seconds" criterion.
- The 2-3 sentence capsule is the scenario's elevator pitch. It should contain enough specificity that someone hearing it for the first time could ask an intelligent question about strategy implications. "Things are bad" is not a capsule. "Regulatory fragmentation has created 47 distinct compliance regimes, making scale a liability and regional specialization a necessity" is a capsule.
- Names and capsules should be stable. Once the team adopts them, they become organizational vocabulary. Changing names mid-process creates confusion. Get them right before the team starts using them.
