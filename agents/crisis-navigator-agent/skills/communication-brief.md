# Communication Brief

Generate stakeholder-specific crisis communications: what to say, to whom, when, and through which channel.

## Prompts Used

1. **assess-situation** - Establish the factual basis for all communications
2. **draft-comms** - Generate audience-specific messages for each stakeholder group

## Workflow

[Assess Situation] -> factual baseline -> [Draft Comms] -> stakeholder-specific messages + channel plan + approval flags

## When to Use

- Multiple stakeholder groups need to be informed about an active crisis and each needs a different message
- The first external communication needs to be drafted and you need to get it right the first time
- Internal and external messaging has become inconsistent and trust is eroding because audiences are hearing different stories
- The crisis is resolved and you need to send a final update to all stakeholder groups
- A stakeholder has heard about the crisis through informal channels and you need to get ahead of the narrative

## What You Get

1. A factual situation summary that serves as the single source of truth for all communications
2. Drafted messages for each stakeholder group (internal team, leadership, customers, partners, public) at the appropriate detail level
3. PACE channel plan for each audience: primary, alternate, contingency, and emergency communication channels
4. Approval flags indicating which messages require human review before sending
5. A consistency check verifying that all messages tell the same story at different levels of detail

## Example Input

"Database partially restored, 30% of API requests still failing. Three enterprise clients have P1 tickets. Engineering is investigating connection pool configuration. Estimated resolution: 2-4 hours. Stakeholders: internal engineering team (12 people, Slack), CTO and VP Engineering (phone), three enterprise clients (dedicated account managers), all customers (status page + email), no media involvement."

## Tips

- Draft the customer message first. If you cannot explain the situation clearly to an external audience, you have not internalized it well enough to communicate internally either.
- Never send "everything is fine" when it is not. Customers and employees can tell, and the trust damage from downplaying exceeds the trust damage from honest disclosure.
- Include a specific next-update time in every message. "We will provide an update by 4:00 PM ET" is a commitment that demonstrates control. Missing that commitment is worse than the crisis itself.
- External communications (customer-facing, public) always require human approval. The Crisis Navigator drafts, humans approve and send.
